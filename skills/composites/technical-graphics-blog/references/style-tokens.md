## Napkin AI prompt template (use when NAPKIN=true)

```
[DIAGRAM TYPE] for a technical blog post showing [what it depicts].

Theme: [Light/Dark]. Clean and minimal. Background: [ACTIVE BG].
No decorative elements. Editorial aesthetic.
Font: Inter. Minimum 13px for all text. Bold (weight 600) for all primary node labels.
Text on dark fills: white (#FFFFFF).
Text on light tint fills: darkest shade of the same color ramp.

Primary:   [ACCENT1 name + hex]
Secondary: [ACCENT2 name + hex]
Accent:    [ACCENT3 name + hex]

Layout direction: [top-to-bottom / left-to-right]

Nodes:
- [Label]: [what this node represents]

Connections:
- [Node A] → [Node B]: [edge label if needed]

Special:
- Node style: rx=[value]
- Arrow style: [L-bend / bezier / straight]
- [Swimlanes / zones / decision diamonds if applicable]
```

Only generate Napkin prompts for:
✓ Flowcharts and pipeline diagrams
✓ Architecture and sequence diagrams
✓ Hero / cover images
✓ Infographics that include a diagram component

Do NOT generate for:
✗ Bar charts, stat panels, comparison tables, pure icon callout grids

## Downloadable HTML file structure

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>[BLOG_TITLE] — Visuals</title>
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap" rel="stylesheet">
  <style>
    * { box-sizing: border-box; margin: 0; padding: 0; }
    body { font-family: Inter, -apple-system, sans-serif; background: #FFFFFF; color: #0F172A; padding: 40px 24px; max-width: 800px; margin: 0 auto; }
    .visual-section { margin-bottom: 60px; page-break-inside: avoid; }
    .visual-label { font-size: 11px; font-weight: 600; color: [ACCENT1]; text-transform: uppercase; letter-spacing: 0.08em; margin-bottom: 8px; }
    .visual-title { font-size: 16px; font-weight: 600; color: #0F172A; margin-bottom: 16px; }
    svg, .infographic { width: 100%; }
    @media print { .visual-section { page-break-after: always; } }
  </style>
</head>
<body>
  <h1 style="font-size:24px;font-weight:700;margin-bottom:8px">[BLOG_TITLE]</h1>
  <p style="font-size:13px;color:#64748B;margin-bottom:48px">Visual set generated by blog-visual-generator skill</p>

  <!-- Repeat for each visual: -->
  <section class="visual-section">
    <p class="visual-label">Visual [N] — [section name]</p>
    <p class="visual-title">[visual title]</p>
    [SVG or HTML infographic content here]
  </section>

</body>
</html>
```

Save to: `/mnt/user-data/outputs/blog-visuals-[url-slug].html`
Present via `present_files` tool after all visuals are shown inline.
