# Lead Enrichment

Enrich leads extracted by the `github-repo-signals` skill with contact information, professional profiles, company intelligence, and ICP qualification scoring.

## When to Use

- User has an xlsx from the github-repo-signals skill and wants to enrich it
- User asks to find emails, phone numbers, or LinkedIn URLs for GitHub leads
- User wants company intelligence (headcount, funding, industry) for lead companies
- User asks to qualify or score leads against their ICP
- User says "enrich", "find emails", "find contacts", "qualify leads", or similar

## Prerequisites

- An xlsx file generated by `github-repo-signals` skill (or any xlsx with a "Users" sheet containing a "username" column)
- Python 3.9+ with `openpyxl`, `requests`, and optionally `python-dotenv`
- API key for the chosen provider in `.env` or environment

## Phase 1: Assess Data Gaps

Before doing anything, read the xlsx and understand what's there and what's missing.

### Step 1: Read the xlsx

Open the xlsx file and analyze the Users sheet. Compute:

- **Total users** in the sheet
- **Have public email**: count of rows where `email` is non-empty
- **Have commit email**: count of rows where `commit_email` is non-empty
- **Have any email**: union of the above
- **Have company**: count of rows where `company` is non-empty
- **Have location**: count of rows where `location` is non-empty
- **Have blog/website**: count of rows where `blog` is non-empty
- **Have twitter**: count of rows where `twitter` is non-empty
- **Unique companies**: distinct non-empty company names
- **Multi-repo users**: count where `total_repos_interacted` > 1

Present this as a data gaps summary table to the user.

### Step 2: Ask for Provider Preference

Present the user with enrichment provider options:

> "How would you like to enrich this data?
>
> 1. **SixtyFour** — email finding ($0.05/lead professional, $0.20/lead personal), lead enrichment (job title, LinkedIn, seniority), company intel (headcount, funding, industry), and ICP qualification scoring
> 2. **Apollo** — bulk email/phone lookup, firmographic data, LinkedIn matching
> 3. **Other** — tell me what provider you'd like to use and I'll help set it up
>
> I currently have a tool built for SixtyFour. Apollo and other providers can be added."

If the user picks **SixtyFour**, proceed to Phase 2.
If the user picks **Apollo**, let them know the tool isn't built yet — offer to build it or proceed manually.
If the user picks **Other**, ask for details and help them set it up.

## Phase 2: Plan Enrichment (SixtyFour)

### Step 3: Recommend Enrichment Actions

Based on the data gaps from Step 1 AND the user's company context (from the github-repo-signals skill's Phase 3), recommend which enrichment actions to run and in what order.

**Available actions and when to recommend each:**

| Action | When to Recommend | Cost |
|--------|-------------------|------|
| `find-email` | When >50% of leads lack email — this is usually the #1 gap | $0.05/lead (professional) or $0.20/lead (personal) |
| `enrich-lead` | When you need job titles, LinkedIn, seniority to identify decision-makers | Check SixtyFour pricing dashboard |
| `enrich-company` | When company names exist but you need firmographic data (headcount, funding, industry) to qualify | Check SixtyFour pricing dashboard |
| `qa-agent` | When the user has clear ICP criteria and wants automated scoring | Check SixtyFour pricing dashboard |

**Decision framework:**

1. **User wants outbound sales leads:**
   - If email coverage < 40%: Start with `find-email`
   - If company coverage > 30%: Run `enrich-company` to get firmographic data, then `enrich-lead` to find decision-makers
   - If user has clear ICP: Finish with `qa-agent` to score and prioritize

2. **User wants to identify companies (account-based approach):**
   - Start with `enrich-company` for all unique companies
   - Then `enrich-lead` on users at the most promising companies
   - `find-email` only for the final shortlist

3. **User wants quick wins:**
   - Start with `find-email` on the top 50 by interaction_score
   - Review results before scaling up

Ask the user:

> 1. Proceed with all recommended actions
> 2. Start with just [cheapest/quickest action] to test quality first
> 3. Adjust the plan (change limits, skip certain actions, etc.)"

**Do NOT proceed until the user confirms.** Credits cost money.

### Step 6: Run Enrichment

Execute the appropriate action(s) using `tools/enrich_sixtyfour.py`.

**For find-email:**
```bash
python3 tools/enrich_sixtyfour.py \
    --input .tmp/repo_signals.xlsx \
    --action find-email \
    --mode PROFESSIONAL \
    --limit 50 \
    --output .tmp/repo_signals_enriched.xlsx
```

**For enrich-lead (with custom fields):**
```bash
python3 tools/enrich_sixtyfour.py \
    --input .tmp/repo_signals.xlsx \
    --action enrich-lead \
    --limit 50 \
    --struct '{"current_title": "Current job title", "linkedin_url": "LinkedIn profile URL", "seniority": "Seniority level: junior, mid, senior, lead, director, vp, c-level", "years_experience": "int: Estimated years of professional experience", "tech_focus": "Primary technology focus area"}' \
    --output .tmp/repo_signals_enriched.xlsx
```

Customize the `--struct` fields based on what the user actually needs. The struct values are plain-English descriptions that tell SixtyFour what to look for. Supported types: `str` (default), `int`, `float`, `bool`, `list`, `list[str]`, `dict`. Prefix the description with the type if not string (e.g., `"int: Number of employees"`).

**For enrich-company:**
```bash
python3 tools/enrich_sixtyfour.py \
    --input .tmp/repo_signals.xlsx \
    --action enrich-company \
    --limit 50 \
    --struct '{"headcount": "int: Number of employees", "funding_stage": "Latest funding stage (seed, series A, B, C, etc.)", "total_funding": "Total funding raised in USD", "industry": "Primary industry", "description": "One-line company description", "website": "Company website URL"}' \
    --output .tmp/repo_signals_enriched.xlsx
```

**For qa-agent:**
```bash
python3 tools/enrich_sixtyfour.py \
    --input .tmp/repo_signals.xlsx \
    --action qa-agent \
    --limit 50 \
    --criteria '[{"criteria_name": "ICP Fit", "description": "B2B SaaS company with 50-500 employees building real-time communication products", "weight": 0.7, "threshold": 0.5}, {"criteria_name": "Decision Maker", "description": "Person is a senior engineer, engineering manager, or above", "weight": 0.3}]' \
    --output .tmp/repo_signals_enriched.xlsx
```

Customize `--criteria` based on the user's actual ICP. Each criterion needs:
- `criteria_name`: Short label
- `description`: Detailed description of what qualifies
- `weight`: Importance weight (0-1, must sum to ~1)
- `threshold` (optional): Minimum score to pass

**Important notes:**
- Always start with a small `--limit` (e.g., 50) for the first run so the user can validate quality
- If running multiple actions sequentially, use the output of the previous step as input to the next
- The tool adds new columns to the xlsx — it does NOT overwrite existing data

## Error Handling

- **401 (Invalid API key):** Check that `SIXTYFOUR_API_KEY` is set correctly in `.env`
- **429 (Rate limited):** The tool auto-retries. If persistent, wait 1 minute and try again.
- **Async timeout (15 min):** SixtyFour enrichment can be slow for complex queries. Try reducing batch size.
- **Low hit rates:** SixtyFour needs enough context to find people. If only a GitHub username is available with no name/company, hit rates will be low. Run `enrich-lead` first to get more context, then `find-email`.
